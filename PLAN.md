# PLAN — 오늘의 냉장고 레시피 3개 (MVP)

## 1) 목표
- 사용자가 **텍스트로 재료**를 입력하면
- **15분 내 가능한 레시피 3개**를 추천하고
- 부족한 재료를 **장보기 리스트**로 정리
- 장보기는 **쿠팡 파트너스 링크**로 연결

## 2) 타겟/톤
- 타겟: 자취(1인가구) + 신혼부부
- 톤: 초간단, 실패 확률 낮게, 도구/설거지 최소

## 3) MVP 범위
### MUST
- 재료 텍스트 입력
- 옵션: 시간(10/15/20), 1~2인분, 조리도구, 제외재료/알레르기
- 레시피 3개 + 장보기 리스트 생성
- 결과 permalink(공유 링크)
- 기본 로깅(요청/응답 메타)

### SHOULD (v1.5)
- 자주 있는 기본양념(소금/후추/간장/식용유) 토글
- 결과 저장(익명 세션) + 최근 5개 히스토리

### COULD (v2)
- 소셜 로그인 + 즐겨찾기/주간 플랜
- 사진 인식
- 영양/칼로리 고도화
- 쿠팡 상품 매칭 추천(검색링크 → 추천상품)

## 4) 페이지 구성
- `/` 입력 폼
- `/r/{id}` 결과 페이지 (레시피 3개 + 장보기)
- (선택) `/me` 히스토리/저장

## 5) 핵심 품질 전략
- LLM 출력은 **반드시 JSON 스키마 고정**
- 서버에서 **검증 후 재시도** (시간/금지재료/형식)

## 6) 마일스톤(2주 MVP)
### M1 — Skeleton (D1~D2)
- Next.js UI 틀 + FastAPI 골격
- API 스키마 확정
- DB 최소 테이블 준비

### M2 — Recommend API (D3~D6)
- LLM 연동 + 검증/재시도
- 결과 저장/조회
- 프론트 결과 렌더

### M3 — Shopping + Coupang (D7~D10)
- 장보기 리스트 생성/렌더
- 쿠팡 파트너스 링크 생성
- 공유 링크 완성

### M4 — Polish/Deploy (D11~D14)
- 프롬프트 튜닝(테스트 30~50케이스)
- 에러/로딩 UX
- 배포: Front(Vercel) + Back(Render)

## 7) 의사결정 메모
- 프론트: **Next.js 추천** (공유 링크/SEO/서버 연동 편함)
- 백엔드: FastAPI
- 배포: Render(백), Vercel(프론트)
